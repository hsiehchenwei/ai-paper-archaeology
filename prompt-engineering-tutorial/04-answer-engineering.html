Chapter 04

答案工程

AI 說的「好」不等於標籤上的「正面」。如何將語言模型的發散思維，收斂為精確的答案？

很多人以為 Prompt 寫好就結束了，但其實「答案」本身也需要設計。論文稱之為 Answer Engineering。這是一個將「自由文本」轉化為「結構化輸出」的關鍵過程。

1. 答案空間 (Answer Space) vs 標籤空間 (Label Space)

"Answer engineering aims to search for an answer space Z and a map to the original output Y that results in an effective predictive model."

"In manual design, the space of potential answers Z and its mapping to Y are crafted manually..."

兩個空間的映射

假設你要做情感分類，你的標籤（Label Space, Y）只有兩個：Positive 和 Negative。

但如果你問 AI：「這部電影怎麼樣？」，它可能會回答："Fantastic", "Great", "Terrible", "Boring"... 這些詞構成了答案空間（Answer Space, Z）。

Answer Engineering 的任務就是建立這兩者之間的橋樑。你需要告訴系統：「如果 AI 說 'Fantastic'，那就等於 'Positive'。」

2. 為什麼這很重要？

"In many cases, the answer space Z is the space of all tokens... In these cases, it is most common to directly map answer z to the final output y using the identity mapping."

無約束 vs 有約束

在翻譯任務中，Answer Space 是無限的（所有可能的句子）。但在分類任務中，Answer Space 必須被嚴格限制。

如果不做 Answer Engineering，你的 AI 系統就會變得很不可控。例如你期待它輸出 "True" 或 "False"，結果它輸出了 "Maybe" 或 "Correct"，這會導致你的程式碼無法解析結果。

現代使用對照

這就是為什麼你需要 "JSON Mode"。

在使用 ChatGPT API 時，你可能用過 response_format={"type": "json_object"}，或者在 Prompt 裡寫：「請只回答 Yes 或 No，不要有多餘解釋」。

這些操作的本質，全都是 Answer Engineering。你在強迫 AI 的 Answer Space 收斂到你定義的 JSON Schema 或特定關鍵字上，以便你的程式能正確讀取。這篇論文在 2021 年就將這種「約束輸出」的行為理論化了。

3. 答案空間設計 (Answer Space Design)

決定了形狀後，接下來的問題是：我們該選擇哪些詞作為答案？

A. Manual Design (手工設計)

最直觀的方法。人類覺得哪個詞好，就用哪個。

  • Unconstrained Spaces: 不限制答案。例如生成任務，所有詞彙表裡的詞都是潛在答案。
  • Constrained Spaces: 限制答案範圍。例如分類任務，我們規定只能輸出 "Positive" 或 "Negative"。

B. Automated Search (自動化搜索)

這部分是論文的精華之一。研究者發現，人類定義的標籤（Label）往往不是模型最喜歡的。

"Discrete Answer Search... It is possible that manually created answers are sub-optimal... We define a function that generates a paraphrased set of answers... The probability of the final output is then defined as the marginal probability."

離散答案搜索 (Discrete Answer Search)

如果模型對 "Excellent" 的響應比 "Positive" 更強烈，為什麼我們要強迫它用 "Positive"？

論文提出了幾種自動尋找最佳答案詞的方法：

  1. Answer Paraphrasing (答案改寫)：利用回譯（Back-translation）技術，把標籤翻譯成其他語言再翻回來，產生多個候選詞（如 "Great", "Good", "Nice"），然後讓模型自己選它最喜歡的那個。
  2. Prune-then-Search (先剪枝再搜索)：先從詞彙表中篩選出高頻詞，再計算這些詞與目標類別的關聯度，找出最佳代表詞（Verbalizer）。
  3. Label Decomposition (標籤拆解)：對於複雜標籤如 per:city_of_death，直接拆解成 person, city, death 三個詞來計算機率。
現代使用對照

從 Verbalizer 到 DSPy / Optimization。

在 GPT-3/4 時代，模型對自然語言的理解能力極強，我們通常不再需要去微調 "Positive" 還是 "Great" 這種細節詞彙，直接用人類直覺的詞彙通常效果就不錯。

但是，這個「自動優化」的概念演變成了現代的 Prompt Optimization 框架（如 DSPy）。我們不再只優化單個單詞，而是讓 AI 自動優化整個 Prompt 結構和範例，這本質上就是 Answer Engineering 與 Template Engineering 的集大成者。

"Continuous Answer Search... Very few works explore the possibility of using soft answer tokens which can be optimized through gradient descent... assign a virtual token for each class label."

連續答案搜索 (Continuous Answer Search)

這是一個更大膽的想法：為什麼答案必須是人類語言？

我們可以為每個類別訓練一個「虛擬向量」(Virtual Token)。這個向量不需要對應任何英文字，它只是高維空間中的一個點，專門用來代表這個類別。這就像是讓 AI 發明自己的語言來標記答案。
