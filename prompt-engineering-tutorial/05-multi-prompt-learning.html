Chapter 05

多重提示學習

獨奏不如合奏。當單一 Prompt 無法解決問題時，我們需要集成、增強與分解。

在前面的章節，我們假設只給 AI 一個 Prompt。但現實世界的問題往往很複雜，單一句話說不清楚。這時候，我們需要引入「多重提示」（Multi-Prompt Learning）。這不僅是技術的堆疊，更是 AI 思維的升級。

1. 提示增強 (Prompt Augmentation)

"Prompt augmentation, also sometimes called demonstration learning, provides a few additional answered prompts that can be used to demonstrate how the LM should provide the answer..."

"For example, instead of just providing a prompt of 'China's capital is [Z].', the prompt can be prefaced by a few examples..."

示範學習 (Demonstration Learning)

這就是我們現在常說的 Few-Shot Prompting。與其讓 AI 憑空猜測，不如先給它看幾個正確的例子。

論文指出，這種方法利用了語言模型學習「重複模式」的能力。

現代使用對照

這是 Chain-of-Thought (CoT) 的前身。

雖然這篇論文發表時 CoT (2022) 還沒正式提出，但 Prompt Augmentation 的概念已經包含了 CoT 的種子。現在當你在 Prompt 裡寫：「Let's think step by step: First... Second...」，其實就是在做 Prompt Augmentation，你給了 AI 一個「思考過程」的範例，讓它模仿。

2. 提示集成 (Prompt Ensembling)

"Prompt ensembling is the process of using multiple unanswered prompts for an input... This sort of prompt ensembling can leverage the complementary advantages of different prompts."

多數決 (Majority Voting)

如果一個 Prompt 可能會出錯，那就用五個不同的 Prompt 問同一個問題，然後看大多數的答案是什麼。

例如：想知道某句話的情感，可以分別問：
1. "情感是？"
2. "這句話讓人感覺？"
3. "評價是正面還是負面？"
然後綜合這三個結果。

現代使用對照

這就是 Self-Consistency。

現在的高級 Prompting 技巧中，有一種叫 "Self-Consistency CoT"，就是讓模型生成多條推理路徑，然後選出現頻率最高的答案。這其實就是 Prompt Ensembling 的變體。

3. 提示分解 (Prompt Decomposition)

"One intuitive method to address this problem is to break down the holistic prompt into different sub-prompts, and then answer each sub-prompt separately."

分而治之 (Divide and Conquer)

對於複雜任務（如命名實體識別 NER），直接問「把所有人名都找出來」可能很難。不如拆解成：「這段話裡有提到『張三』嗎？是人名嗎？」「有提到『台北』嗎？是地點嗎？」。

現代使用對照

這就是 AI Agents 的雛形。

現在的 Agent (如 AutoGPT, LangChain) 工作流程，本質上就是將一個大目標（例如「幫我寫個貪吃蛇遊戲」）拆解成無數個小 Prompt（「先寫遊戲邏輯」、「再寫畫面渲染」、「最後寫控制輸入」）。這就是 Prompt Decomposition 的極致表現。
